# 内联面板定位 规格说明

## 目的
将区块详情面板定位到被点击区块的正右侧，而非固定在轨道右边缘，包含边界检测和自适应方向。

## 需求

### 需求：面板必须出现在被点击区块的相邻位置
当用户点击一个 track-pill 区块时，详情弹窗面板的左边缘必须紧贴被点击区块的右边缘，水平间距为 8px。面板的垂直顶部必须与被点击区块在 rail 中的顶部对齐。

#### 场景：点击最左列的光照区块
- **当** 用户点击一个光照区块（定位在 left: 8%, width: 24%）
- **则** 面板的左边缘应位于距 rail 左边缘约 32% + 8px 处，面板顶部与区块在 rail 中的顶部偏移一致

#### 场景：点击中间列的咖啡因区块
- **当** 用户点击一个咖啡因区块（定位在 left: 38%, width: 24%）
- **则** 面板的左边缘应位于距 rail 左边缘约 62% + 8px 处

#### 场景：点击最右列的睡眠区块
- **当** 用户点击一个睡眠区块（定位在 left: 68%, width: 24%）
- **则** 面板的左边缘应位于距 rail 左边缘约 92% + 8px 处，受边界裁剪约束

### 需求：面板必须保持在轨道边界内
面板不得超出 continuous-rail 容器的右边缘。当计算位置会导致溢出时，面板必须被裁剪以适应 rail 边界。

#### 场景：最右列区块的右侧溢出
- **当** 用户点击最右列的区块且计算得到的 left + panelWidth 超出 rail 宽度
- **则** 面板的 left 必须被裁剪为 railWidth - panelWidth - 8px，确保面板在 rail 内完全可见

#### 场景：面板纵向位置靠近轨道底部
- **当** 用户点击靠近时间轴底部的区块
- **则** 面板的 top 必须调整，使面板不超出 rail 的总高度

### 需求：面板定位必须使用动态像素计算
面板的水平位置必须通过被点击元素的实际渲染位置（使用 `getBoundingClientRect()` 或 `offsetLeft`/`offsetWidth`）计算，而非硬编码的 CSS 百分比值。

#### 场景：窗口缩放导致区块位置变化
- **当** 视口缩放导致区块的像素位置发生变化
- **则** 下一次点击区块时，面板必须基于新的渲染位置进行定位

### 需求：点击天标签也必须使用内联面板定位
点击 day-chip 按钮时，面板也必须出现在被点击 day-chip 的相邻位置，使用与 track-pill 区块相同的内联定位逻辑。

#### 场景：点击天标签按钮
- **当** 用户点击一个 day-chip 按钮
- **则** 面板应出现在 day-chip 的右侧，遵循相同的 8px 间距和边界裁剪规则
